# アプリアイコンのサイズと仕様

## 推奨サイズ

- **icon.png**（メインアイコン）: **1024×1024 px**、正方形必須
- PNG形式、透過背景を推奨

## Android アダプティブアイコンではみ出す場合

Android のアダプティブアイコンは **108dp × 108dp** のキャンバスで、実際にマスク内に収まるのは **中央 72dp × 72dp**（約 **66%**）の「セーフゾーン」だけです。  
端まで描いたデザインだと、マスクで切り抜かれて **はみ出して見える** ことがあります。

### 対処方法

1. **前景用に専用画像を用意する（推奨）**
   - **1024×1024 px** の PNG で、**ロ
   ゴを中央 66% 程度に収めて** 描く
   - 周囲は **透過** にし、余白（パディング）を入れる
   - このファイルを `assets/adaptive-icon.png` として保存し、`app.json` の `android.adaptiveIcon.foregroundImage` で指定する

2. **app.json の指定**
   - 共通アイコン: `icon` → `./assets/icon.png`
   - Android のみ前景を変える: `android.adaptiveIcon.foregroundImage` → `./assets/adaptive-icon.png`
   - 前景用に余白付きの `adaptive-icon.png` を用意すれば、はみ出しを防げます。

### 参考（Android の仕様）

- 前景レイヤー: 108dp × 108dp
- セーフゾーン（マスク内で確実に表示）: 中央 72dp × 72dp
- 1024px 画像でいうと、中央おおよそ **682×682 px** がセーフゾーン相当

### 画像の確認ポイント

- 解像度が **1024×1024** か（縦横がずれていないか）
- ロゴが **中央 66% 以内** に収まっているか
- 前景画像は **透過 PNG** にしているか（背景色は `backgroundColor` で指定）
